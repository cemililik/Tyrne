### Review-round arc on PR #28 — five rounds, agent-driven verification compounded

The pattern that emerged: agent-driven re-reads compound in value when each round names a different *axis* (correctness, security, doc-discipline, link-integrity). The first round catches obvious bugs; later rounds catch second-order effects of the first round's fixes (round 2's deferred `BlockMapped` made round 4's audit-log clarification load-bearing; round 4's `.agents/skills/` consolidation made round 5's `t-018.md` link drift visible). The cost is real (six review-round commits) but each commit is small (median diff ≈ 50–150 lines) and the result is a final merge with no known open issues at the bot-reviewer layer.

**Discipline note for future PRs:** five rounds is high — three is the typical previous-PR ceiling (PRs #23, #26 both had three). The extra two rounds came from (a) the cross-cutting skills migration that wasn't planned, and (b) the maintainer asking for a fresh-eyes self-review after R2. Both are legitimate triggers, not noise. A future PR with similar surface (kernel object + cap-gated wrappers + activation hook) should budget three rounds and treat additional rounds as discovery-driven rather than ordinal.

### The §Simulation discipline carried through ADR-0028's bootstrap-AS-wrap special case

ADR-0028 row 0 captures a load-bearing decision: the bootstrap address space is wrapped via `Mmu::wrap_existing_root` rather than `Mmu::create_address_space`. The latter would either (a) skip cap minting for the live topology (the cap table stays empty until first `cap_create_address_space`, but the bootstrap AS *is* a live topology that needs a cap to be reachable from the userspace-bound capability surface) or (b) double-zero-fill the in-use L0 frame (the `create_address_space` contract requires the input frame to be zero-filled; the live L0 frame contains valid descriptors and zeroing it mid-execution would unmap the kernel from itself).

The §Simulation table caught both failure modes before code lands. The implementation hit neither. The pattern of "non-obvious row 0 catches the load-bearing setup decision before downstream rows" is now visible at three ADR data points (ADR-0026 idle-current self-yield; ADR-0027 SCTLR.M=1 transition; ADR-0028 bootstrap-wrap special case). The rule that emerged from B1's smoke-regression arc is now stable.

### The "leak-path closure" discipline matured during T-018's review-round arc

T-018's `cap_create_address_space` originally allocated PMM frames *before* the cap-table insertion that could fail with `DerivationTooDeep`. The review-round arc surfaced the leak path (commit `8b9f52e`'s "Fix 1" depth-preflight refinement) and closed it with a step-2c depth preflight that runs *before* PMM alloc. The final preflight chain — lookup → kind → DERIVE → no-widening → depth → arena.is_full → table.is_full → THEN pmm.alloc_frame → create_as → arena.alloc → cap_derive — has no leak path under v1's single-core cooperative semantics because every fallible step happens before the unrecoverable PMM commitment.

The discipline is broader than T-018: any kernel-object creation path that consumes a PMM frame must preflight every rejectable check before the commit. The pattern is now load-bearing for B4 (initial userspace image — needs `cap_create_address_space` for the userspace AS + `cap_map` for each segment) and B5 (`MemoryRegion` cap variant — will introduce a second PMM-consuming creation path).

### Bot review-round signal quality remained consistent with B1 + B2 calibration

Same pattern as the B1 and B2 closure observations: factual-mechanical findings (path drift, stale links, missing barriers, doc-vs-code mismatch, missing test counts) had ~95 % apply rate across PR #25 / #26 / #27 / #28. Architectural / refactor findings ("extract this helper", "use `for` instead of `while`", "split this function") had ~50 % apply rate; the project's CLAUDE.md "minimum-surface" discipline filtered most over-engineering preferences. **One new observation:** when CodeRabbit's "AI-Agents prompt" embeds specific path claims (e.g., "the canonical is `docs/analysis/<type>-reviews/`"), the agent should verify before applying — the prompt's *intent* was right (the SKILL.md drift was real) but the *target path* was wrong (the actual canonical is `docs/analysis/reviews/<type>-reviews/`). The lesson: trust CodeRabbit's *finding direction* more than its *fix prescription*; the prescription often needs adjustment for repo-specific conventions.

### The B2 closure adjustment "Codify ADR §Simulation row → host-test or audit-log line" remained open through B3
